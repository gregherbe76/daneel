ÉTAPE 2 — IMPLÉMENTATION (après mon OK)

A. Logique de ranking modifiée

   Dans le step shortlist_generation :

   1. Récupérer les candidats du job avec leurs matching_score 
      existants (logique actuelle inchangée)
   
   2. Pour chaque candidat, joindre le dernier 
      technical_evaluations row si présent (basé sur run_id ou 
      candidate_id selon le pattern)
   
   3. Calculer le bonus :
      - Si pas de technical_evaluation OU evaluated=false OU 
        overall_score est null : bonus = 0
      - Sinon : bonus = (overall_score / 100) × 20
   
   4. Calculer final_score = min(100, matching_score + bonus)
   
   5. Trier par final_score DESC
   
   6. Stocker dans la shortlist (ou retourner) chaque candidat avec :
      - matching_score (base)
      - codematch_overall (raw, ou null)
      - bonus_applied (0 à 20)
      - final_score (matching + bonus, plafonné à 100)
      - tech_evaluated (boolean : true si bonus > 0)

B. Schéma BDD

   Si la table shortlists existe et stocke les ranks :
   
   - Ajouter les colonnes : 
     - matching_score (numeric, nullable)
     - codematch_overall (numeric, nullable)

     - final_score (numeric, nullable)
     - tech_evaluated (boolean, default false)
   
   - Migration via drizzle-kit
   
   Si la shortlist est calculée à la volée à chaque affichage
