SECTION B — Extend SourcingProvider (TypeScript)
============================================================

B.1. Nouveau fichier : 
     artifacts/api-server/src/routes/workflows/providers/extend.ts

B.2. Implémente AgentProvider interface (signature découverte 
     dans diagnostic Étape 1) :
     
     class ExtendSourcingProvider implements AgentProvider {
       readonly id: number;
       readonly name: string;
       readonly type = "extend";
       
       constructor(
         id: number, 
         name: string, 
         apiKey: string,        // ex_xxx, déjà déchiffrée
         config: { baseUrl?: string | null } | null
       )
       
       async run(input: AgentProviderRunInput): Promise<SourcingRunResult>
       
       async validateConnection(): Promise<{ ok: boolean; error?: string }>
     }

B.3. Méthode validateConnection() :
     - GET {baseUrl}/v1/me avec Bearer apiKey
     - Timeout 8s
     - Mapping :
       * 200 → { ok: true } (clé valide, peu importe le tier)
       * 401 → { ok: false, error: "Invalid API key" }
       * 5xx → { ok: false, error: "Extend API unreachable" }
       * timeout → { ok: false, error: "Extend API timeout" }

B.4. Méthode run() — orchestration async polling :

     1. Récupérer profile_urls depuis input.payload.job.exampleProfileUrls
        - Si vide ou null : retourner 
          { candidates: [], stats: { error: "no_profile_urls" } }
        - Si > 10 : tronquer à 10 (l'API Extend rejette > 10)

     2. Préparer le body Extend :
        {
          profile_urls: array,
          job_context: input.payload.job.description ?? undefined,
          search_location: input.payload.job.location ?? undefined,
          max_results: 20,
          min_results: 3,
          scoring_criteria: input.payload.job.mustHaveSkills ?? undefined
        }

     3. POST {baseUrl}/v1/find-similar avec Bearer apiKey
        - Timeout 8s sur ce POST (juste pour recevoir 202)
        - Si pas 202 : map error et retourner result avec error

     4. Récupérer analysis_id depuis la réponse 202

     5. Polling loop :
        - Délai initial 30s (pipeline ~60-150s en moyenne)
        - Puis poll GET /v1/find-similar/:id toutes les 10s
        - Timeout total côté Daneel : 12 minutes (Extend a un 
          watchdog 10 min côté serveur, on prend une marge)
        - Si status === "running" : continue polling
        - Si status === "completed" : extract candidates et break
        - Si status === "failed" : retourner avec error
        - Si timeout 12 min : retourner avec error: "extend_timeout"

     6. Mapper Extend candidates → Daneel SourcingCandidate[] :
        Pour chaque candidate Extend :
        {
          name: extend.name,
          headline: "",  // Extend ne fournit pas
          location: input.payload.job.location ?? "",  // copie du job
          currentCompany: "",  // Extend ne fournit pas
          email: null,
          linkedinUrl: extend.linkedinUrl,  // camelCase Extend OK
          githubUrl: "",
          username: null,
          confidence: extend.score / 10,  // Extend score 0-10 → 
                                          // Daneel confidence 0-1
          emailSource: null,
          skills: [],  // Extend ne fournit pas par candidat
          summary: extend.scoreReason ?? "",  // bullets concaténés
          evidence: extend.linkedinUrl,  // pour anti-fabrication
          potentialRisks: "",
          source: "Extend pattern-match"
        }

     7. Retourner :
        {
          candidates: mapped[],
          stats: {
            extend_analysis_id: analysisId,
            extend_pattern_title: pattern.title,
            extend_total_found: total_found,
            extend_below_minimum: below_minimum,
            extend_polling_duration_ms: duration
          }
        }

B.5. Headers Bearer apiKey + User-Agent: "Daneel/1.0 (Extend 
     provider)" sur tous les appels.

B.6. Logger via SimpleLogger injecté (pattern existant), pas 
     req.log direct.

with async polling"
